# TabTypist

An open-source system-wide AI typing assistant that suggests inline ghost-text completions as the user types in any text field. Differentiates from existing alternatives by supporting non-English languages (Amharic first) and lower-end hardware via a hosted inference tier.

## Language

**Completion**:
The proposed text the model produces, shown as ghost text after the caret.
_Avoid_: Suggestion, ghost text, prediction (use **Completion** in user-facing copy and code).

**Acceptance**:
The act of committing a **Completion** into the focused text field, via Tab.
_Avoid_: Confirm, insert, apply.

**Partial Acceptance**:
The act of committing one word of a multi-word **Completion** via Tab, leaving the remainder visible as ghost text at the updated caret position. Does not notify Rust or trigger new inference; the remaining ghost text is repositioned by the next AX poll.
_Avoid_: Partial confirm, partial insert.

**Dismissal**:
The act of rejecting an unaccepted **Completion**, via Escape or by continuing to type something that no longer matches.
_Avoid_: Cancel, reject.

**Trigger**:
The event that causes TabTypist to request a new **Completion** from the model. Currently per-keystroke with debounce.
_Avoid_: Invocation, fire.

**Local mode**:
**Completions** generated by a model running on the user's machine. Privacy-default, no network. The only mode available offline.
_Avoid_: On-device, offline mode.

**Hosted mode**:
**Completions** generated by a model running on TabTypist's hosted inference service. Higher quality on low-end hardware and for non-English languages; requires the user to be online and opted-in.
_Avoid_: Cloud mode, server mode, online mode.

**Default model**:
The model TabTypist installs and routes to for a given language without the user choosing one explicitly. Determined by the languages selected at onboarding.
_Avoid_: Built-in model, system model.

**Model override**:
A user-selected model that replaces the **Default model** for a language, exposed under "Advanced" in onboarding and settings.

**Exclusion list**:
The set of apps and field types where TabTypist does not generate **Completions**. Has three tiers: always-off (no override), default-off (user can re-enable), and default-on with optional per-app disable.
_Avoid_: Blocklist, denylist (use **Exclusion list** in user-facing copy).

## Relationships

- A user selects one or more languages at onboarding; each selected language has a **Default model** and an optional **Model override**.
- A **Trigger** produces zero or one **Completion**, which the user either accepts (**Acceptance**) or dismisses (**Dismissal**).
- A **Completion** is generated in either **Local mode** or **Hosted mode**; the mode is a per-user setting, not per-completion.

## Example dialogue

> **Dev:** "When the user types into a Slack DM in Amharic, does the **Completion** come from the **Default model** for Amharic, or from **Hosted mode**?"
> **Product:** "It depends on the user's mode setting. In **Local mode**, the Amharic **Default model** runs on-device. In **Hosted mode**, the request goes to our service, which may use a larger model. The user picks the mode; we don't switch silently."

## Flagged ambiguities

- "Suggestion" and "ghost text" were used interchangeably with **Completion** in early discussion — canonical term is **Completion**.
